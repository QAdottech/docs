**Preview environments** are temporary, isolated environments created for specific feature branches or pull requests. They allow you to test changes in a production-like environment before merging code into your main branch.

## What are Preview Environments?

Preview environments are dynamically created environments that:

- Deploy your application with the latest changes from a specific branch
- Provide a unique URL for testing the changes
- Are automatically cleaned up after the feature is merged or the branch is deleted
- Enable stakeholders to review and test features before they go live

## Integration with API

You can trigger test runs against preview environments using the [**Start Run API**](/api-reference/start-run). The API allows you to override environment configurations per application when starting a test run.

## Integration with CI/CD

You can test preview environments from any CI/CD pipeline using the QA.tech REST API. The API supports:

- **Dynamic URL configuration** - Override environment URLs using `applications` parameter
- **Multiple application testing** - Test frontend, backend, and other apps simultaneously
- **Blocking mode** - Wait for test completion before proceeding (poll run status)
- **PR/MR-based deployments** - Automatic testing on pull requests or merge requests
- **Any CI/CD platform** - Works with GitLab, GitHub Actions, Bitbucket, Azure DevOps, CircleCI, Jenkins, and more
