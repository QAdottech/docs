# QA.tech API Reference

The QA.tech API allows you to programmatically interact with our platform, enabling you to create tests, run them, and retrieve results. This comprehensive reference covers all endpoints, authentication methods, error handling, and provides practical examples.

## Base URL

All API requests should be made to:

```
https://api.qa.tech/v1
```

## Authentication

All API endpoints require authentication using Bearer tokens. You can find your API key in the QA.tech dashboard under **Project Settings â†’ Configuration**.

### API Key Authentication

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
     https://api.qa.tech/v1/projects
```

### JavaScript Example

```javascript
const response = await fetch('https://api.qa.tech/v1/projects', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  }
});
```

### Python Example

```python
import requests

headers = {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
}

response = requests.get('https://api.qa.tech/v1/projects', headers=headers)
```

## Rate Limits

The API implements rate limiting to ensure fair usage:

- **Standard Plan**: 100 requests per minute
- **Professional Plan**: 500 requests per minute
- **Enterprise Plan**: 1000 requests per minute

Rate limit headers are included in all responses:

```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1640995200
```

## Error Handling

### HTTP Status Codes

| Code | Description |
|------|-------------|
| 200 | Success |
| 201 | Created |
| 400 | Bad Request - Invalid parameters |
| 401 | Unauthorized - Invalid API key |
| 403 | Forbidden - Insufficient permissions |
| 404 | Not Found - Resource doesn't exist |
| 429 | Too Many Requests - Rate limit exceeded |
| 500 | Internal Server Error |

### Error Response Format

```json
{
  "error": {
    "code": "INVALID_PARAMETER",
    "message": "The 'url' parameter is required",
    "details": {
      "parameter": "url",
      "value": null
    }
  }
}
```

### Common Error Codes

| Code | Description |
|------|-------------|
| `INVALID_API_KEY` | API key is invalid or expired |
| `RATE_LIMIT_EXCEEDED` | Too many requests in time period |
| `RESOURCE_NOT_FOUND` | Requested resource doesn't exist |
| `INVALID_PARAMETER` | Required parameter missing or invalid |
| `INSUFFICIENT_PERMISSIONS` | User lacks required permissions |
| `INTERNAL_ERROR` | Server-side error occurred |

## Projects

### List Projects

Retrieve all projects for the authenticated user.

**Endpoint:** `GET /projects`

**Parameters:**
- `page` (optional): Page number for pagination (default: 1)
- `limit` (optional): Number of projects per page (default: 20, max: 100)

**Response:**

```json
{
  "data": [
    {
      "id": "proj_123456789",
      "name": "E-commerce Website",
      "url": "https://example.com",
      "created_at": "2024-01-15T10:30:00Z",
      "updated_at": "2024-01-15T10:30:00Z",
      "status": "active",
      "test_count": 15,
      "last_test_run": "2024-01-15T09:00:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 45,
    "pages": 3
  }
}
```

**Example:**

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
     "https://api.qa.tech/v1/projects?page=1&limit=10"
```

## SDKs and Libraries

### JavaScript/Node.js SDK

Install the official JavaScript SDK:

```bash
npm install @qa-tech/sdk
```

**Usage:**

```javascript
import { QATech } from '@qa-tech/sdk';

const client = new QATech('YOUR_API_KEY');

// List projects
const projects = await client.projects.list();

// Create a test

  name: 'Login Test',
  description: 'Test user login functionality',
  instructions: 'Navigate to login page, enter credentials, and verify successful login'

// Run a test
const run = await client.runs.create('proj_123456789', 'test_123456789');
```

## Interactive Examples

### Complete Test Workflow

This example demonstrates a complete workflow for creating and running tests:

```javascript
import { QATech } from '@qa-tech/sdk';

const client = new QATech('YOUR_API_KEY');

async function completeTestWorkflow() {
  try {
    // 1. Get or create a project
    const projects = await client.projects.list();
    let project = projects.data[0];
    
    if (!project) {
      project = await client.projects.create({
        name: 'My Test Project',
        url: 'https://example.com'
      });
    }

    // 2. Create a test
    const test = await client.tests.create(project.id, {
      name: 'E-commerce Checkout Test',
      description: 'Test the complete checkout process',
      instructions: `
        1. Navigate to the homepage
        2. Search for a product
        3. Add the first product to cart
        4. Proceed to checkout
        5. Fill in shipping information
        6. Complete the purchase
        7. Verify order confirmation
      `
    });

    // 3. Run the test
    const run = await client.runs.create(project.id, test.id);
    console.log(`Test run started: ${run.id}`);

    // 4. Poll for completion
    let completed = false;
    while (!completed) {
      const status = await client.runs.get(project.id, test.id, run.id);
      if (status.data.status === 'completed') {
        completed = true;
        console.log(`Test completed with result: ${status.data.result}`);
        
        if (status.data.result === 'failed') {
          console.log('Screenshot URL:', status.data.screenshot_url);
          console.log('Video URL:', status.data.video_url);
        }
      } else {
        await new Promise(resolve => setTimeout(resolve, 5000)); // Wait 5 seconds
      }
    }

  } catch (error) {
    console.error('Error:', error.message);
  }
}

completeTestWorkflow();
```

### Error Handling Example

```javascript
import { QATech } from '@qa-tech/sdk';

const client = new QATech('YOUR_API_KEY');

async function handleErrors() {
  try {
    const projects = await client.projects.list();
  } catch (error) {
    switch (error.code) {
      case 'INVALID_API_KEY':
        console.error('Please check your API key');
        break;
      case 'RATE_LIMIT_EXCEEDED':
        console.error('Rate limit exceeded. Please wait before retrying.');
        break;
      case 'RESOURCE_NOT_FOUND':
        console.error('The requested resource was not found');
        break;
      default:
        console.error('An unexpected error occurred:', error.message);
    }
  }
}
```

## Best Practices

### API Key Security

- Store API keys securely and never commit them to version control
- Use environment variables for API keys in production
- Rotate API keys regularly
- Use different API keys for different environments

### Rate Limiting

- Implement exponential backoff for retries
- Monitor rate limit headers in responses
- Cache responses when appropriate
- Batch requests when possible

### Error Handling

- Always check HTTP status codes
- Implement proper error handling for all API calls
- Log errors for debugging
- Provide user-friendly error messages

### Testing

- Use the API to automate your testing workflows
- Implement webhooks for real-time notifications
- Set up monitoring for test results
- Use the SDKs for easier integration
