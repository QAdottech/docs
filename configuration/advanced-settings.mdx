# Advanced Configuration

QA.tech provides extensive configuration options for advanced users who need fine-grained control over AI agent behavior, project settings, and integration capabilities. Learn how to optimize your AI-powered testing setup.

## Project Settings

### Core Project Configuration

Access your project settings in the QA.tech web interface:

<Steps>
  <Step title="Navigate to Settings">
    Go to app.qa.tech and select your project
    Click on the "Settings" icon in the left sidebar
    Select "General" under Project settings
  </Step>
  <Step title="Configure Project Details">
    Update your project name and description
    Set project visibility and access controls
    Configure project-specific settings
    Save your changes
  </Step>
  <Step title="Manage Applications">
    Add your application URLs
    Configure environment settings
    Set up authentication if needed
    Test application connectivity
  </Step>
</Steps>

### Application Configuration

Configure your applications for AI testing:

```json
{
  "applications": {
    "main-app": {
      "url": "https://your-app.com",
      "environment": "production",
      "authentication": {
        "type": "username_password",

        "password": "secure_password"
      }

    "staging-app": {

      "environment": "staging",
      "authentication": {
        "type": "username_password",
```

### GitHub Action Configuration

```yaml
name: QA.tech Tests

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
    - name: Run QA.tech Tests
      uses: QAdottech/run-action@v2
      with:
        project_id: ${{ secrets.QATECH_PROJECT_ID }}
        api_token: ${{ secrets.QATECH_API_TOKEN }}
        test_plan_short_id: 'jgbinp'
        blocking: true
```

### GitLab CI Integration

Configure GitLab CI/CD integration:

```yaml
stages:
  - test

qa-tech-tests:
  stage: test
  image: node:18
  before_script:
    - echo "QA.tech tests will be executed via GitHub Actions"
  script:
    - echo "Using QA.tech cloud-based testing platform"
  environment:
    name: staging
  variables:
    QA_TECH_ENVIRONMENT: staging
    QA_TECH_TIMEOUT: 60000
    QA_TECH_RETRY_COUNT: 3
```
